# Codex 任务 08：Final Polish & QA v4.0

## 任务定位

这是 Zen Canvas UI/UX v4 的最终发布验收阶段。

本阶段不再大改信息架构，而是系统性解决：

- 状态完整性；
- 视觉一致性；
- 跨平台；
- 小窗口和缩放；
- 无障碍；
- 性能；
- 文案；
- 动效；
- 旧视觉债务；
- 发布质量。

---

## 开始前

必须确认任务 01–07 均已单独提交并通过阶段验收。

创建发布验收清单，不要在一个大提交中混入新的业务功能。

---

|---|---:|

## 允许修改

全前端范围内的：

- 视觉一致性；
- 文案；
- 无障碍；
- 动效；
- 响应式；
- 性能；
- 测试；
- 设计文档；
- 删除确认无引用的旧兼容层。

后端仅允许修复被测试证明的 UI 集成缺陷，并必须单独说明；不得顺手新增业务能力。

---

## 1. 全局状态矩阵

每个核心页面检查：

- 初始；
- 加载；
- 空；
- 正常；
- 进行中；
- 成功；
- 部分成功；
- 失败；
- 权限不足；
- 取消；
- 可恢复；
- 不可恢复。

不得以通用 Toast 替代页面内必要状态。

建立状态清单并逐项截图。

---

## 2. 视觉一致性审计

检查：

- 是否存在硬编码颜色；
- 是否存在 Tailwind 固定调色板；
- 是否存在随机圆角；
- 是否存在卡片套卡片；
- 是否存在内容区毛玻璃；
- 是否存在多个 Primary；
- 是否存在重复安全 Notice；
- 是否存在不统一的按钮高度；
- 是否存在文字低于规范；
- 是否存在非品牌图标风格。

若旧兼容导出已无引用，可安全删除；不能为了“清理代码”破坏已验收页面。

---

## 3. 跨平台验收

### Windows

- 原生窗口控制；
- 关闭 Hover；
- 资源管理器文案；
- 125%/150% 缩放；
- 普通非 Retina 显示；
- Win11 字体渲染；
- NSIS 安装包。

### macOS

- 交通灯；
- 拖拽区；
- 访达文案；
- Retina；
- Dark Mode；
- 键盘快捷键使用 Cmd。

不得用同一外壳强行伪装两平台。

---

## 4. 窗口和缩放

至少测试：

- 1440×900；
- 1280×800；
- 980×680；
- 1920×1080；
- Windows 125%；
- Windows 150%；
- macOS Retina。

检查：

- Sidebar 不遮挡；
- Inspector 可收缩或合理隐藏；
- Footer 不覆盖；
- Dialog 不超出窗口；
- 中文不截断；
- 表格列可用；
- Primary 保持可见。

---

## 5. 键盘验收

全局：

- Ctrl/Cmd+K；
- Esc；
- Tab/Shift+Tab；
- Enter；
- Space；
- 方向键。

文件库：

- 多选；
- Shift 连选；
- Quick Look；
- 右键菜单键盘替代。

整理：

- 接受、保留、修改目标；
- 批量模式。

所有快捷键：

- 不抢占输入框；
- 可发现；
- 可本地化；
- 有焦点恢复。

---

## 6. 无障碍

- WCAG AA 文本对比；
- 控件和焦点至少 3:1；
- 不只靠颜色；
- 所有 Icon Button 有名称；
- Dialog Focus Trap；
- 错误与字段关联；
- Live Region 不重复播报；
- Reduced Motion；
- Windows High Contrast 基础可用；
- 200% 文本缩放不破坏关键任务。

运行自动化检查，并进行人工键盘测试。

---

## 7. 动效

统一使用 Design Token。

允许：

- 选中态；
- Inspector 切换；
- Spotlight；
- 进度；
- 完成反馈；
- 品牌分析动效。

禁止：

- 大量列表飞入；
- 持续脉冲；
- 无意义旋转；
- 大幅缩放；
- 布局抖动。

Reduced Motion 下：

- 移除非必要位移；
- 进度仍可理解；
- 不影响功能。

---

## 8. 性能

记录并优化：

- 首屏渲染；
- Spotlight 打开；
- 文件库滚动；
- Inspector 切换；
- 大量建议；
- 长历史；
- 主题切换。

要求：

- 不因单项选择重渲染整个大列表；
- 搜索请求可取消/防旧结果；
- 无明显内存泄漏；
- 关闭 Dialog 清理监听器；
- Store selector 精确；
- 不引入大体积无必要依赖。

---

## 9. 文案审计

全局删除或替换：

- 工程术语；
- 中英混杂；
- `Unknown`；
- `Smart Matches`；
- Batch/Token/JSON；
- 含义不明的百分比；
- 夸张的隐私绝对化文案。

统一语气：

- 简短；
- 清楚；
- 不责怪用户；
- 说明发生什么；
- 说明下一步；
- 说明能否恢复。

---

## 10. 安全回归

必须人工和自动验证：

- 扫描不移动文件；
- 建议不自动执行；
- Preview 前不可执行；
- 不覆盖；
- 清理进入回收站；
- 恢复有冲突保护；
- AI 云端边界真实；
- 权限错误不降级为危险操作；
- 原生 confirm 已全部移除。

---

## 11. 测试与 CI

最终必须：

```bash
npm run typecheck
npm test
npm run build
```

此外：

- 不允许 `.only` / `.skip` 遗留；
- 不允许测试捕获后返回空对象掩盖模块失败；
- 关键行为使用运行时测试，而不只源码字符串；
- 视觉契约测试不锁死无意义类名；
- CI 状态清楚；
- 构建产物不入库。
