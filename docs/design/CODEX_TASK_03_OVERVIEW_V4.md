# Codex 任务 03：Overview v4.0

## 任务定位

将当前“空间扫描页”重构为真正的 Zen Canvas 概览首页。

概览不是扫描仪表盘，也不是快速操作控制中心。它只回答：

1. 当前文件空间有什么值得注意；
2. 哪一件事最需要用户处理；
3. Zen Canvas 正在后台做什么。

扫描继续存在，但成为概览中的任务，而不是整个首页的产品定义。

1. 将主导航“概览”渲染为新的 Overview；
2. 保留内部 `scanner` 路由标识也可以，不要求本阶段大范围重命名 View；

   - 一个当前最重要任务；

## 1. 页面结构

### A. 当前优先任务

首页最上方显示一个任务，不显示多个竞争主按钮。

优先级建议：

1. 执行中或失败的任务；
2. 需要人工确认的整理建议；
3. 可安全清理空间；
4. 尚未建立索引；
5. 索引需要更新；
6. 一切正常。

示例：

- 有 18 项需要你确认；
- 扫描正在进行，已处理 2,340 个文件；
- 12 项可安全清理，预计释放 6.8 GB；
- 还没有建立本地索引；
- 文件空间保持有序。

### B. 空间摘要

使用一条关联性文案，而不是三到五张统计卡：

> 今天 21:42 扫描了下载、桌面和文档，共分析 12,842 个文件；找到 42 项整理建议和 6.8 GB 可释放空间。

必要时可拆为最多 3 个轻量指标，但不使用大卡片网格。

### C. 最近活动

展示真实操作：

- 扫描完成；
- 生成建议；
- 文件移动/重命名；
- 清理；
- 恢复；
- 部分失败。

每条可点击进入详情；可撤销时显示撤销入口。

### D. 后台任务

只在存在任务时显示：

- 后台索引；
- AI 分析；
- 整理预览生成；
- 执行任务。

无后台任务时，不展示大面积空卡。

## 2. 唯一 Primary 规则

Primary 根据当前状态动态变化：

| 状态 | Primary |
|---|---|
| 未建立索引 | 开始扫描 |
| 扫描中 | 查看扫描进度 |
| 有待确认建议 | 查看建议 |
| 有清理候选 | 查看清理候选 |
| 有部分失败 | 查看失败项 |
| 一切正常 | 扫描新位置 / 检查更新（二选一） |

以下动作使用 Secondary/Ghost/Menu：

- 选择文件夹；
- 重新扫描；
- 暂停；
- 取消；
- 打开文件库；
- 打开设置。

## 3. 扫描任务设计

### 启动扫描

点击“开始扫描”后使用 Sheet 或内嵌任务面板：

1. 默认用户空间；
2. 选择文件夹；
3. 说明只建立本地索引；
4. 开始。

不要跳到另一个信息密集的“扫描仪表盘”。

### 扫描中

显示：

- 当前路径；
- 已处理数量；
- 已用时间；
- 暂停/取消；
- 可安全离开页面。

不要使用虚假的百分比；只有后端能提供可靠总量时才显示进度百分比。

### 扫描完成

显示：

- 实际扫描文件数；
- 新增/更新数量；
- 发现建议数；
- 失败/跳过数量；
- Primary 自动转为下一步。

### 扫描失败/取消

- 明确失败原因；
- 已完成索引是否保留；
- 重试；
- 查看详情；
- 不丢失已有结果。

---

## 4. 数字语义

必须明确：

- `待确认` 是否包含在 `全部建议` 中；
- `可自动整理` 与 `待确认` 的关系；
- `可释放空间` 来自哪些候选；
- `已分析百分比` 的分母是什么。

如果无法准确解释某个百分比，就不显示它。

禁止再使用：

- 含义不明的健康度圆环；
- “参考磁盘容量”作为主指标；
- 当前页杂乱比例作为首页核心数字。

首次打开只展示：

- 本地优先和不会自动移动的短说明；
- Primary：开始扫描；

- 无文件；
- 无待办；
- 扫描中；
- 扫描暂停；
- 扫描取消；
- 扫描失败；
- 部分完成；
- 权限不足；
- AI 关闭；
- AI 配置失败；
- 后台任务失败；
- 当前范围不存在。

- 当前优先任务可以使用一块主要 Content/Raised 区；
- 最近活动使用列表；

- 品牌动效只用于扫描/分析阶段；
- 不持续大面积发光；
- 不使用大型圆环；
